# RecycleBot RL — Mathematical Formulation

A human-guided, value-aware robotic waste-sorting system where the robot learns which **item / bin / robot** action maximizes long-run recovered value, while staying close to human-expert behaviour and asking for help only when useful.

The general system is a partially observable MDP because the robot does not know the true material or pick outcome with certainty. The MVP is simpler: a rolling-horizon contextual bandit with a convex LP assignment inside each decision window.

---

## 1. Problem framing

At each control step, the system observes:

\[
S_t=(\mathcal{W}_t,\mathcal{R}_t,\mathcal{B}_t,H_t,E_t),
\]

where:

- \(\mathcal{W}_t=\{\tau_{1(t)},\ldots,\tau_{n(t)}\}\) is the visible waste-item set;
- \(\tau\) is one tracked trash item;
- \(\mathcal{R}_t\) is the robot-state set;
- \(\mathcal{B}_t\) is the bin-state set;
- \(H_t\) is the operator state;
- \(E_t\) is the stream/environment state.

For each item \(\tau\), the system estimates:

\[
\mathbf{F}_{\tau(t)}\in\mathbb{R}^p.
\]

A single elementary action is:

\[
a_t=a(\tau,r,b),
\]

meaning: pick item \(\tau\) with robot \(r\) and place it into bin \(b\).

The action space includes the null action:

\[
\mathbb{A}_t
=
\{a(\tau,r,b)\mid \tau\in\mathcal{W}_t,\ r\in\mathcal{R}_t,\ b\in\mathcal{B}_t\}
\cup
\{\varnothing\}.
\]

The null action means the item is ignored for pick-and-place in the current decision step.

---

## 2. Action output: trajectory instruction

The mathematical action is not just a label. It produces an executable robot instruction:

\[
\Gamma_t(a(\tau,r,b))
=
\left(
\boldsymbol{\xi}_{\tau,r,b,t},
\mathrm{PICK}(\tau),
\mathrm{PLACE}(b)
\right),
\]

where:

- \(\boldsymbol{\xi}_{\tau,r,b,t}\) is the robot trajectory;
- \(\mathrm{PICK}(\tau)\) is the instruction to grasp the tracked item;
- \(\mathrm{PLACE}(b)\) is the instruction to place it into the target bin.

For the null action:

\[
\Gamma_t(\varnothing)=\mathrm{NOOP}.
\]

---

## 3. Long-run objective

The goal is not to pick the highest-value visible object at this instant. A high-value but hard-to-grasp item can block the robot from picking several easier, moderately valuable items.

The plant objective is therefore steady-state recovered value per unit time:

\[
\max_{\pi}\ \rho(\pi)
=
\liminf_{T\rightarrow\infty}
\frac{
\mathbb{E}_{\pi}\left[\sum_{t=0}^{T-1} R(S_t,A_t)\right]
}{
\mathbb{E}_{\pi}\left[\sum_{t=0}^{T-1}\Delta t_t\right]
}.
\]

If every control step has fixed duration \(\Delta t\), this reduces to the usual average-reward form:

\[
\max_{\pi}\ \rho(\pi)
=
\liminf_{T\rightarrow\infty}
\frac{1}{T}
\mathbb{E}_{\pi}\left[\sum_{t=0}^{T-1} R(S_t,A_t)\right].
\]

---

## 4. General POMDP form

The true environment is partially observable. The robot sees observations, but it does not directly know:

- true material identity;
- exact contamination risk;
- future pick success;
- latent operator attention state;
- downstream verification outcome.

General formulation:

\[
\mathcal{M}_{\mathrm{POMDP}}
=
(\mathcal{S},\mathbb{A},\Omega,P,O,R,\gamma).
\]

At time \(t\):

\[
S_t=(\mathcal{W}_t,\mathcal{R}_t,\mathcal{B}_t,H_t,E_t).
\]

The observation is:

\[
\Omega_t=(O_t^{(1)},O_t^{(2)}),
\]

with:

- \(O_t^{(1)}\): object-detection stream;
- \(O_t^{(2)}\): human hand-tracking / correction stream.

The practical workaround is to use a belief state. For material identity:

\[
\mathbf{b}_{\tau(t)}(M)
=
P(M_\tau=M\mid z_{\tau(t)}).
\]

The belief vector becomes part of \(\mathbf{F}_{\tau(t)}\).

---

## 5. MVP contextual bandit form

For the MVP, collapse the horizon to a one-step decision. The system sees a context \(S_t\), selects one elementary action \(a_t\), observes a reward, and updates the value model.

\[
\max_{\theta}\
\mathbb{E}_{S_t,\ a_t\sim\pi_\theta(\cdot\mid S_t)}
\left[R(S_t,a_t)\right].
\]

This is a contextual bandit because the action is chosen from the current visible state only.

Use the full MDP/RL version later when the following become first-class:

- multi-robot timing;
- robot busy/free queues;
- conveyor congestion;
- missed-pick opportunity cost over multiple windows;
- bin full/empty dynamics;
- human workload allocation.

---

## 6. Item feature vector

Each tracked item \(\tau\) has a feature vector:

\[
\mathbf{F}_{\tau(t)}=
\left[
\mathbf{F}^{\mathrm{pose}}_{\tau(t)},
\mathbf{F}^{\mathrm{shape}}_{\tau(t)},
\mathbf{F}^{\mathrm{condition}}_{\tau(t)},
\mathbf{F}^{\mathrm{material}}_{\tau(t)},
\mathbf{F}^{\mathrm{grasp}}_{\tau(t)},
\mathbf{F}^{\mathrm{economic}}_{\tau(t)},
\mathbf{F}^{\mathrm{human}}_{\tau(t)}
\right].
\]

### 6.1 Minimal item attributes

| Group | Symbol | Meaning |
|---|---|---|
| Track id | \(\mathrm{id}_{\tau}\) | persistent object id across frames |
| Pose | \(P_{\tau,t}\) | simplified pose, e.g. \((x,y,\theta)\) |
| Shape | \(\sigma_{\tau,t}\) | one of \(\{\mathrm{bottle},\mathrm{tray},\mathrm{can},\mathrm{box}\}\) |
| Condition | \(\kappa_{\tau,t}\) | one of \(\{\mathrm{clean},\mathrm{crumpled},\mathrm{occluded}\}\) |
| Material belief | \(P(M_\tau=M\mid z_{\tau,t})\) | probability over \(\mathcal{M}\) |
| Time to exit | \(t^{\mathrm{exit}}_{\tau(t)}\) | time before item leaves reachable workspace |
| Grasp candidates | \(\mathcal{G}_{\tau(t)}\) | candidate grasp set |
| Best grasp score | \(G^*_{\tau(t)}\) | best candidate grasp quality |
| Pick probability | \(p^{\mathrm{pick}}_{\tau,r,t}\) | probability robot \(r\) picks item successfully |
| Place probability | \(p^{\mathrm{place}}_{\tau,r,b,t}\) | probability item lands in target bin \(b\) |
| Human prior | \(\mu_H(a(\tau,r,b)\mid S_t)\) | human-ghost preference signal |

### 6.2 Environment state

The simplified environment state is:

\[
E_t=(U_{1,t},v_t^{\mathrm{belt}}),
\]

where:

- \(U_{1,t}\): lighting condition;
- \(v_t^{\mathrm{belt}}\): conveyor speed.

Sensor contamination, humidity, and temperature are intentionally excluded from the simplified MVP.

### 6.3 Bin state

Each material has a correct bin:

\[
B:\mathcal{M}\rightarrow\mathcal{B}.
\]

For this version, each bin is only marked:

\[
\mathrm{state}(b)\in\{\mathrm{empty},\mathrm{full}\}.
\]

A full bin is removed from the legal action set. Downstream quality metrics can be added later, but the current formulation does not use a bin-quality constraint.

## 7. Expected material-bin gain

Use per-item value, not value per kg.

Let:

\[
V_M^{\mathrm{item}}
\]

be the net recovered value for correctly sorting one item of material \(M\).

The expected gain of placing item \(\tau\) in bin \(b\) is:

\[
G_{\tau,b,t}
=
\sum_{M\in\mathcal{M}}
P(M_\tau=M\mid z_{\tau,t})
\left[
\mathbf{1}[b=B(M)]V_M^{\mathrm{item}}
-
C^{\mathrm{contamination}}_{M,b}
\right].
\]

Here \(B(M)\) is the correct bin for material class \(M\). The indicator term:

\[
\mathbf{1}[b=B(M)]
\]

prevents the model from crediting recovered value when the object is placed into the wrong bin.

If material \(M\) belongs in bin \(b\), then:

\[
C^{\mathrm{contamination}}_{M,b}\approx 0.
\]

If material \(M\) contaminates bin \(b\), then:

\[
C^{\mathrm{contamination}}_{M,b}>0.
\]

---

## 8. Expected one-action net value

The expected net value of one elementary action is:

\[
\widehat{g}_{\tau,r,b,t}
=
 p^{\mathrm{pick}}_{\tau,r,t}
 p^{\mathrm{place}}_{\tau,r,b,t}
 G_{\tau,b,t}
-
C^{\mathrm{risk}}_{\tau,r,b,t}
-
C^{\mathrm{motion}}_{\tau,r,t}.
\]

This is better than raw item value because it includes:

- probability of successful pick;
- probability of successful placement;
- contamination cost;
- motion/cycle cost;
- risk cost.

A useful greedy display score is value density:

\[
A_{\tau,r,b,t}
=
\frac{\widehat{g}_{\tau,r,b,t}}{c^{\mathrm{cycle}}_{\tau,r,t}+\epsilon}.
\]

This attractiveness score is only a heuristic. The LP planner keeps time in a capacity constraint instead of dividing the objective by cycle time.

---

## 9. Human-ghost policy

The human hand-tracking and correction stream is used to estimate:

\[
\mu_H(a\mid S_t)
=
P(\text{operator would choose action }a\mid S_t).
\]

It must be defined over the same action set as the robot policy:

\[
\mathbb{A}_t
=
\{a(\tau,r,b)\}\cup\{\varnothing\}.
\]

The null action matters. If the human ghost has no probability on \(\varnothing\), the KL policy is biased toward always picking something.

Treat \(\mu_H\) as a behavioural prior, not as ground truth. Operators are skilled, but they are still inconsistent, fatigued, and attention-bound.

---

## 10. KL-regularized policy

Let \(\widehat{Q}(S,a)\) be the learned action-value estimate. The robot chooses a policy close to the human prior while still favouring high-value actions:

\[
\pi^*
=
\arg\max_{\pi\in\Delta(\mathbb{A}_t)}
\left[
\sum_{a\in\mathbb{A}_t}\pi(a\mid S_t)\widehat{Q}(S_t,a)
-
\beta
D_{\mathrm{KL}}\left(
\pi(\cdot\mid S_t)\ \|\ \mu_H(\cdot\mid S_t)
\right)
\right].
\]

Closed form:

\[
\pi^*(a\mid S_t)
=
\frac{
\mu_H(a\mid S_t)\exp(\widehat{Q}(S_t,a)/\beta)
}{
\sum_{a'\in\mathbb{A}_t}
\mu_H(a'\mid S_t)\exp(\widehat{Q}(S_t,a')/\beta)
}.
\]

Interpretation:

- \(\beta\rightarrow\infty\): imitate the human ghost;
- \(\beta\rightarrow 0^+\): become value-greedy;
- decrease \(\beta\) as the value model becomes more reliable.

Support warning:

If \(\mu_H(a\mid S_t)=0\), then the forward-KL solution assigns \(\pi^*(a\mid S_t)=0\). To explore outside human demonstrations, mix \(\mu_H\) with a small uniform prior or test the reverse-KL variant.

---

## 11. Reverse-KL variant

The reverse-KL variant is:

\[
\max_{\pi\in\Delta(\mathbb{A}_t)}
\sum_a\pi_a\widehat{Q}_a
+
\beta\sum_a\mu_a\log\pi_a.
\]

The KKT solution is:

\[
\pi_a^*
=
\frac{\beta\mu_a}{\nu-\widehat{Q}_a},
\qquad
\nu>\max_a\widehat{Q}_a,
\]

with \(\nu\) chosen so that:

\[
\sum_a\frac{\beta\mu_a}{\nu-\widehat{Q}_a}=1.
\]

Forward KL is cleaner for the MVP. Reverse KL is useful only if coverage beyond the strongest human modes becomes a measured problem.

## 12. Short-horizon LP assignment

The LP planner is used when more than one robot can act in the same decision window.

Decision variable:

\[
y_{\tau,r,b,t}\in[0,1].
\]

A value near 1 means: assign item \(\tau\) to robot \(r\) and bin \(b\) in this planning window.

Objective:

\[
\max_y
\sum_{\tau\in\mathcal{W}_t}
\sum_{r\in\mathcal{R}_t}
\sum_{b\in\mathcal{B}_t}
 y_{\tau,r,b,t}\widehat{g}_{\tau,r,b,t}.
\]

### C1 — at most one assignment per item

\[
\sum_{r\in\mathcal{R}_t}
\sum_{b\in\mathcal{B}_t}
y_{\tau,r,b,t}
\leq 1
\qquad\forall \tau\in\mathcal{W}_t.
\]

### C2 — robot capacity per decision window

\[
\sum_{\tau\in\mathcal{W}_t}
\sum_{b\in\mathcal{B}_t}
c^{\mathrm{cycle}}_{\tau,r,t}y_{\tau,r,b,t}
\leq C_{r(t)}
\qquad\forall r\in\mathcal{R}_t.
\]

### C3 — reachability / bin availability

The implementation removes illegal actions before the LP is solved:

\[
y_{\tau,r,b,t}=0
\quad\text{if}\quad
\tau\text{ is unreachable by }r
\quad\text{or}\quad
b\text{ is full}.
\]

There is no bin-quality constraint in the simplified version. Wrong-bin risk is handled directly by \(G_{\tau,b,t}\) through the no-credit indicator and the contamination cost.

## 13. Reward decomposition

For an executed action set \(A_t\), the reward can be written as:

\[
R_t
=
R_t^{\mathrm{value}}
-
R_t^{\mathrm{contamination}}
-
R_t^{\mathrm{missed}}
-
R_t^{\mathrm{robot}}
-
R_t^{\mathrm{human}}.
\]

Value:

\[
R_t^{\mathrm{value}}
=
\sum_{a(\tau,r,b)\in A_t}
\mathbf{1}[b=B(M_\tau)]V_{M_\tau}^{\mathrm{item}}.
\]

Contamination:

\[
R_t^{\mathrm{contamination}}
=
\sum_{a(\tau,r,b)\in A_t}
C^{\mathrm{contamination}}_{M_\tau,b}.
\]

Missed opportunity:

\[
R_t^{\mathrm{missed}}
=
\sum_{\tau\in\mathrm{valuable\ missed}}
\max_{b\in\mathcal{B}_t}G_{\tau,b,t}.
\]

Robot cost:

\[
R_t^{\mathrm{robot}}
=
\sum_{a(\tau,r,b)\in A_t}
\left(C^{\mathrm{motion}}_{\tau,r,t}+C^{\mathrm{risk}}_{\tau,r,b,t}\right).
\]

Human cost:

\[
R_t^{\mathrm{human}}
=C^{\mathrm{human}}N_t^{\mathrm{interventions}}.
\]

Safety should not be treated only as a soft penalty in deployment. Use a hard action shield or a constrained MDP if humans can be exposed to robot motion.

## 14. Human intervention rule

A human query is useful only when the model is uncertain in a way the human can reduce, and the value at stake justifies the interruption.

Let:

\[
q_{r(t)}\in\{0,1\}
\]

indicate whether robot \(r\) requests supervision.

Operator attention budget:

\[
\sum_{r\in\mathcal{R}_t}q_{r(t)}\leq H_{\max}.
\]

Trigger:

\[
q_{r,t}
=
\mathbf{1}\left[
\max_{a\in\mathbb{A}_t}\mathrm{Var}_{\mathrm{epi}}[\widehat{Q}(S_{r(t)},a)]>\eta_u
\ \land\
\max_{a\in\mathbb{A}_t}\bar{Q}(S_{r(t)},a)>\eta_v
\right].
\]

Both quantities are the robot's own estimates from the Bayesian posterior: the epistemic variance and the posterior-mean value. The value threshold \(\eta_v\) keeps interruptions to decisions that matter. Each intervention subtracts a fixed cost \(C^{\mathrm{intervention}}\) from the reward, and the value model is updated with this net reward.

Use epistemic uncertainty, such as posterior variance or ensemble disagreement. Raw entropy is not enough: some ambiguity is sensor-limited and will not be solved by asking a human.

## 15. Off-policy evaluation

Every logged decision should include the behaviour-policy probability:

\[
\mu_t(a_t\mid S_t).
\]

Logged dataset:

\[
\mathcal{D}_{\mathrm{log}}
=
\{(S_t,a_t,R_t,\mu_t(a_t\mid S_t))\}_{t=1}^{N}.
\]

Self-normalized IPS estimate for a new policy \(\pi_{\mathrm{new}}\):

\[
\widehat{V}_{\mathrm{SNIPS}}(\pi_{\mathrm{new}})
=
\frac{
\sum_t
\frac{\pi_{\mathrm{new}}(a_t\mid S_t)}{\mu_t(a_t\mid S_t)}R_t
}{
\sum_t
\frac{\pi_{\mathrm{new}}(a_t\mid S_t)}{\mu_t(a_t\mid S_t)}
}.
\]

Without \(\mu_t(a_t\mid S_t)\), reliable off-policy evaluation is not possible after the fact.

\(\mu_t\) is the probability under the *executed* decision rule: the sampling probability of the exploration-mixed policy on autonomous steps, and \(\mu_t(a_t\mid S_t)=1\) on intervention steps, where the human override is deterministic. Because intervention steps are deterministic, actions other than the override have zero behavior probability there: SNIPS is reliable for target policies close to the logged behavior and degrades for distant ones (low effective sample size), which the random-policy experiment shows directly.

---

## 16. Tracking and commitment

Once an action is dispatched, the robot must be committed long enough to execute it.

For a committed action \(a(\tau,r,b)\):

\[
t\in[t_c(\tau),t^{\mathrm{complete}}_{\tau,r}].
\]

During that interval, robot \(r\) is unavailable for new assignments.

This avoids LP thrashing, where the planner keeps changing its mind faster than the robot can move.

## 18. Learning loop

1. Observe \(S_t=(\mathcal{W}_t,\mathcal{R}_t,\mathcal{B}_t,H_t,E_t)\).
2. Build legal actions \(\mathbb{A}_t\), including \(\varnothing\).
3. Estimate \(\widehat{Q}(S_t,a)\) for each action.
4. Estimate the human ghost \(\mu_H(a\mid S_t)\).
5. Form the KL-regularized policy \(\pi^*(a\mid S_t)\).
6. Dispatch the selected trajectory instruction \(\Gamma_t(a_t)\).
7. Log reward, action probability, outcome, and any human correction.
8. Update the value model, pick model, placement model, and human-ghost model.
9. Evaluate candidate policies off-policy before deployment.
